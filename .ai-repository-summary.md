## Repository Overview

**Name:** workflow-orchestration-queue-bravo20
**Application:** OS-APOW (Orchestration System for AI-Powered Orchestration Workflows)
**Type:** Headless agentic orchestration platform
**Primary Language:** Python 3.12+

## Purpose

OS-APOW transforms GitHub Issues into autonomous AI-driven development workflows. It acts as a tireless background service that:
- Receives GitHub webhooks
- Queues and claims tasks
- Dispatches AI workers in isolated DevContainers
- Reports status via GitHub labels and comments

## Environment Variables

### Required
- `GH_ORCHESTRATION_AGENT_TOKEN` - GitHub PAT (repo, workflow, project scopes)
- `ZHIPU_API_KEY` - ZhipuAI API key
- `GITHUB_REPO` - Target repository (owner/repo)
- `WEBHOOK_SECRET` - HMAC webhook secret

### Optional
- `POLLING_INTERVAL` - Seconds between polls (default: 60)
- `HEARTBEAT_INTERVAL` - Seconds between heartbeats (default: 300)
- `DEBUG_MODE` - Enable debug logging (default: false)
